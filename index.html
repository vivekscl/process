The OSS-Generic Reference Process

The OSS-Generic Reference Process described in this repo documents common practices among several OSS projects on GitHub, notably, TEAMMATES, SE-EDU projects, PowerPointLabs.

General references

  • Formats and Conventions: Formats and conventions for PRs, Issues, Files, etc.

  • Community Structure: Levels, roles, and responsibilities of dev community members.

For core members

  • Reviewing PRs: Instructions for PR reviewers

  • Merging PRs: Instructions for merging a PR

For lead members

  • Overseeing Issue/PR Life Cycle: Instructions for team/area leads on overseeing issue/PR life cycles.

  • Defining Labels: Instructions for creating labels for the issue tracker.
